# Security Policy for EduMind AI Study Assistant

## Supported Versions

| Version | Supported          |
|---------|------------------|
| `v1.x`  | :white_check_mark: Yes |
| `v0.x`  | :x: No             |

---

## Reporting a Vulnerability

If you discover a security vulnerability in EduMind AI Study Assistant, please **report it responsibly**:

1. **Email:** Send details to **dimpal.yadav2912@gmail.com**.  
2. **Subject Line:** Use `[SECURITY]` in your email subject.  
3. **Include:**  
   - Steps to reproduce the issue  
   - Affected version(s)  
   - Impact assessment (if possible)  

We will **acknowledge your report** within 48 hours and aim to **resolve the issue promptly**.

---

## Security Best Practices for Contributors

When contributing to the EduMind AI Study Assistant repository:

- Do not commit sensitive information like passwords, API keys, or secrets.  
- Always validate and sanitize user input.  
- Keep dependencies up to date to avoid known vulnerabilities.  
- Follow coding standards and review PRs carefully for potential security risks.  

---

## Supported Security Features

- Uses standard GitHub authentication for repository access.  
- Encourages responsible disclosure of vulnerabilities.  
- Promotes safe coding practices for contributors.
